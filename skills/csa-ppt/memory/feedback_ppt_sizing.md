PPT幻灯片内容必须在**横向和纵向都**填满页面95%以上的空间。这是系统级硬性规则。

**Why:** 用户多次反馈生成的PPT底部、侧面有大量空白。仅宽度填满不够，高度也必须填满。

**How to apply:**

## 高度预算（7.5" slide, 必须填到96%+）
生成前必须做精确高度预算计算，确保所有区域加起来 ≥ 7.2"：
- 标题栏: ~0.88" (12%)
- 副标题条: ~0.34" (4.5%)
- 主内容区: ~4.40" (59%) ← 这是最大可伸缩区域
- 蓝色扩展区: ~0.88" (12%)
- Executive Summary栏: ~0.58" (8%)
- 间距合计: ~0.12" (1.5%)
- **底部边距不超过 0.30"**

## 宽度预算（13.33" slide）
- 左右边距各 0.20"（3%）
- 内容区宽度 = 12.93"（97%）
- 卡片/图表必须铺满内容区全宽

## 生成后必须验证
在代码末尾打印实际覆盖率：
```python
print(f'Coverage: {Y/SLIDE_H*100:.1f}%')  # 必须 >= 95%
```
如果 < 95%，必须增大各区域高度直到达标。

## 字体最低标准
- 页面主标题: ≥ 26pt Bold
- 栏目头标题: ≥ 15pt Bold
- 斜体副标题: ≥ 12pt Bold Italic
- 正文/列表: ≥ 11pt
- 图表标签: ≥ 9pt
- 图标/编号: ≥ 24pt
